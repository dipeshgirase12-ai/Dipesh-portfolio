## 🚀 Overview

A **cyber-themed portfolio** website showcasing projects, skills, certifications, and achievements — built with a Matrix/Tron-inspired aesthetic.

| Feature | Description |
|---------|-------------|
| ⚡ **Hero Section** | 3D particle sphere with glitch text & animated counters |
| 🖥️ **Terminal Interface** | Interactive CLI with custom commands |
| 🎯 **Project Cards** | Hover overlays with live demo & GitHub links |
| 🏆 **Certifications** | Filterable, sortable showcase with modal previews |
| 🤖 **AI Chatbot** | Floating assistant answering questions about the developer |
| 🌓 **Theme Toggle** | Dark/Light mode switch |

## 🚀 Deployment

This site is deployed on **Vercel** with continuous deployment from the `main` branch.

| Step | Action |
|------|--------|
| 1 | Push code to `main` branch |
| 2 | Vercel auto-detects changes |
| 3 | Site rebuilds and deploys automatically |

```bash
git add .
git commit -m "Update portfolio"
git push origin main
```

### Vercel Configuration

The site uses a **static HTML/CSS/JS** setup with the following config:

- **Root Directory:** `portfolio/` (where the source files live)
- **Framework:** Other / None
- **Build Command:** Disabled (no build step needed)
- **Output Directory:** `portfolio/` (served directly)

A root `vercel.json` file configures the `portfolio/` folder as the output directory, disables the build step (static site), and adds SPA rewrites so navigation works without 404 errors.
